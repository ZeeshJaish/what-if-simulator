# What If Scenario Simulator

## Overview
What If Scenario Simulator is a React-based web application that creates funny and imaginative alternate-life scenarios. Users can select or enter a person and combine them with an unexpected role, such as "What if Dhoni was a singer?" or "What if Donald Trump was a cricketer?" The app then displays a creative result with a short story, fake achievements, meme-style content, and a visual result card.

The project is designed as an interactive entertainment app while staying within core React and JavaScript concepts.

## Problem Statement
People enjoy humorous "what if" situations, memes, and alternate-reality content, but most of this content is scattered across social media. There is no simple interactive app where users can create their own scenario combinations and instantly see a fun result.

This project solves that by giving users a playful scenario generator where they can explore different combinations, view creative outputs, and save their favorite results.

## Objective
The main objective is to build a React application that allows users to generate, view, and save creative "what if" scenarios using forms, reusable components, conditional rendering, routing, and JavaScript-based logic.

## Key Features
- Landing page with project introduction and sample scenarios
- Scenario generator with person and role inputs
- Predefined scenario suggestions for quick generation
- Creative result page with story, fake records, and fun achievements
- Meme-style image section using static data or a public meme API
- Free limit for the first 5 scenario generations
- Login gate after 5 generations to continue using the app
- Frontend login/session handling using localStorage
- Scenario history for logged-in users
- Responsive layout using CSS or Tailwind CSS

## App Flow
1. User lands on the home page and understands the app concept.
2. User opens the scenario generator.
3. User enters or selects a person and a role.
4. App creates a funny "what if" output using JavaScript logic and predefined content.
5. User can generate up to 5 scenarios without logging in.
6. After 5 scenarios, the app asks the user to login.
7. Logged-in users can continue generating scenarios and view their history.
